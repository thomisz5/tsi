Pizzaria Senado
Aqui tudo acaba em pizza!

Considere que você foi contratado para criar um pequeno sistema com formulário para a realização de pedidos para a Pizzaria Senado, onde tudo acaba em pizza. O formulário deve possuir as seguintes entradas: tamanho da pizza (pequena, média e grande), sabor, preço, opção de borda recheada, opção de entrega em casa ou balcão. Vale considerar que a quantidade de sabor por pizza segue a seguinte relação:

  • Pizza Pequena: 1 sabor - R$ 20,00
  • Pizza Média: 2 sabores - R$ 25,00
  • Pizza Grande: 3 sabores - R$ 30,00

Ainda, o formulário também deve registrar os seguintes dados do cliente: nome, telefone e endereço. O campo de endereço apenas deve ser apresentado no formulário se a opção entrega for marcada. Quando o funcionário clicar no botão enviar, os dados precisam ser armazenados no WebStorage. Ademais, a qualquer momento, o funcionário poderá visualizar a página relatório com a lista de pedidos do dia, ou seja, os pedidos armazenados no WebStorage.

Exigências

  • O pedido de uma pizza deve ser representado por um objeto pizza.
  • O cliente precisa ser representado por um objeto cliente, sendo que o objeto pizza deve ter referência ao objeto cliente.
  • Os pedidos de pizza precisam ser armazenados em um vetor chamado pedidos.
  • O vetor pedidos precisa ser armazenado no WebStorage.
  • O campo telefone precisa ser validado via REGEX e apresentar máscara via algum plugin do JQuery.
  • A página relatório deve listar todos os pedidos de pizza armazenados no WebStorage em uma lista ou tabela, devendo apresentar os seguintes dados por pizza: tamanho, sabores e nome do cliente.
